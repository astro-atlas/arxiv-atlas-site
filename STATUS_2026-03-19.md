## Key Learnings

### What Works Well
- Multi-model pipeline keeps each step focused
- Forcing `pages_created` in manifest prevents ingestion bugs
- Dispute boxes make disagreements explicit instead of hidden
- Cleanup immediately after ingest prevents stale files

## Database Maintenance

### View papers
```sql
SELECT arxiv_id, title FROM papers ORDER BY created_at DESC;
```

### View pages for a paper
```sql
SELECT p.title, pp.page_slug 
FROM papers p 
JOIN paper_page_links ppl ON p.id = ppl.paper_id 
JOIN pages pp ON ppl.page_id = pp.id 
WHERE p.arxiv_id = '2603.15869';
```

### Remove duplicates (if any reappear)
```sql
DELETE FROM pages WHERE page_slug LIKE 'papers/%' AND page_type = 'paper';
DELETE FROM pages WHERE page_type = 'paper' AND title LIKE '2603 %';
```

## Wiki Page Conventions

- **Reference links:** `<a href="#refN">[N]</a>` (regular, comma-separated)
- **Key concepts:** Max 4 per paper infobox
- **Disputes:** `<div class="dispute">` with `.dispute-side` subsections
- **WikiLinks:** `<a href="/pages/slug" class="wikilink">Text</a>`
- **Concept levels:** foundational / intermediate / specialist
